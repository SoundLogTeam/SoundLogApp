# Codex 요청 프롬프트 모음

아래 문장을 복사해서 Codex에게 요청하면 Soundlog 프로젝트 흐름에 맞게 작업하기 쉽습니다.

## 새 기능 구현

```text
Soundlog에서 [기능명]을 구현하고 싶어.
먼저 관련 문서를 읽고 기능 계약, 화면 영향 범위, 상태/API/권한 요구사항을 정리해줘.
엣지케이스가 있으면 나에게 질문하고, 계획 리뷰 후 구현해줘.
구현 뒤에는 타입체크와 웹 확인, 코드 리뷰, 리뷰 반영까지 진행해줘.
```

## 화면 디자인 개선

```text
[화면명]을 더 완성도 있게 개선해줘.
기존 디자인 문서와 현재 컴포넌트 구조를 먼저 확인하고,
어떤 컴포넌트를 수정할지 계획을 세워줘.
텍스트 겹침, 모바일 크기, 빈 상태, 로딩 상태, 에러 상태까지 고려해서 구현해줘.
```

## 자연어 UI 피드백 루프

```text
[화면명]이 [자연어 피드백: 예. 답답해 보여 / 더 고급스럽게 / 버튼이 애매해] 보여.
비개발자 UI 피드백 루프로 진행해줘.
내 말을 구체적인 UI 목표로 번역하고,
계획 -> 계획 리뷰 -> 계획 반영 -> 구현 -> 코드/UI 리뷰 -> 리뷰 반영 -> 검증 순서로 진행해줘.
리뷰에서 나온 문제는 최종 답변 전에 반영해줘.
```

## 버그 수정

```text
현재 앱에서 아래 버그가 발생해.
[에러 메시지 또는 현상]
원인을 추적하고 최소 수정으로 해결해줘.
수정 후 같은 종류의 문제가 다른 파일에도 있는지 확인하고 타입체크까지 해줘.
```

## Mock API 추가

```text
아직 서버가 없는 [기능명]을 테스트할 수 있게 mock-server에 API를 추가해줘.
성공/로딩/실패 상태를 모두 테스트할 수 있게 만들고,
React Query 훅 또는 API facade까지 연결해줘.
```

## 문서 정리

```text
이번에 구현한 내용을 docs에 정리해줘.
비개발자가 이해할 수 있게 기능 목적, 사용 흐름, 관련 파일, 테스트 방법을 포함해줘.
불필요한 임시 문서는 제거해줘.
```

## 커밋과 푸시

```text
변경사항을 점검하고 불필요한 파일이 없는지 확인한 뒤 작업 브랜치에 커밋하고 PR 만들어줘.
커밋 메시지는 한국어로 간결하게 작성해줘.
```
